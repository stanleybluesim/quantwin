2) 用 AS 的“追溯矩阵”直接生成工程 Backlog
-----------------------------

    Architecture Spec (AS) v5.0-7.1…

3) 先做 P0 最小可用闭环（按 RS 的 O1 指标验收）
-------------------------------

实现优先级用 RS 的 O1 来驱动，因为它定义了系统“能不能跑起来”的硬门槛：

*   **KR1**：采集完成到可检索 P95 ≤ 5 分钟
*   **KR2**：查询端到端 P99 ≤ 800ms
*   **KR3**：吞吐 ≥ 100 QPS 且成功率 ≥ 99.5%
*   **KR4**：资源紧张或模块异常时按 L0-L3 自动降级，核心检索与证据链仍可用
    Requirements Spec (RS) v3.1 – Q…

对应到工程切片就是：

*   Ingestion → Indexing → Query 的端到端链路先跑通，并把“可检索”定义落实为可测指标与用例（不要先追求全功能 UI/报告）。
